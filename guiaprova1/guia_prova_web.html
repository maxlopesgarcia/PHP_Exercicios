HTML — Estrutura Básica

HTML é a linguagem de marcação das páginas web. Todo arquivo começa com <!DOCTYPE html>, que avisa ao navegador qual versão do HTML está sendo usada. 

<!DOCTYPE html>
<html lang="pt-br">
<head>
    <!-- Define o charset para acentuação funcionar -->
    <meta charset="UTF-8">

    <!-- Responsividade: adapta para celulares -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <title>Título da Aba</title>

    <!-- Vincula um arquivo CSS externo -->
    <link rel="stylesheet" href="style.css">
</head>
<body>

    <header><!-- cabeçalho --></header>
    <nav><!-- menu de navegação --></nav>
    <main><!-- conteúdo principal --></main>
    <footer><!-- rodapé --></footer>

</body>
</html>
Semântica: usar as tags corretas (header, nav, main, footer) ajuda motores de busca e leitores de tela a entenderem a estrutura da página.

HTML — Formulários

Formulários coletam dados do usuário. O atributo method define como os dados são enviados; action define para onde vão. 

<!--
  method="post"  → dados viajam no corpo da requisição (mais seguro, invisível na URL)
  method="get"   → dados aparecem na URL (?campo=valor)
  action="..."   → para onde os dados são enviados (ex: outra página PHP)
-->
<form action="?abobora=resultado" method="post">

    <!-- label conecta ao input pelo id (acessibilidade) -->
    <label for="nome">Nome:</label>
    <input type="text"     name="nome"      id="nome"><br>

    <label for="email">E-mail:</label>
    <input type="email"    name="email"     id="email"><br>
    <!-- type="email" valida o formato automaticamente no navegador -->

    <label for="senha">Senha:</label>
    <input type="password" name="senha"     id="senha"><br>
    <!-- type="password" oculta os caracteres digitados -->

    <label for="data_nasc">Data de Nascimento:</label>
    <input type="date"     name="data_nasc"  id="data_nasc"><br>
    <!-- type="date" abre um calendário para o usuário escolher -->

    <label for="hora">Hora:</label>
    <input type="time"     name="hora"      id="hora"><br>

    <label for="genero">Gênero:</label>
    <select name="genero" id="genero">
        <!-- cada <option> é um item da lista suspensa -->
        <option value="m">Masculino</option>
        <option value="f">Feminino</option>
        <option value="o">Outro</option>
    </select><br>

    <button type="submit">Enviar</button>

</form>
Lembre-se: o atributo name no input é o que o PHP usa para recuperar o valor ($_POST["nome"]). Sem ele, o campo não chega ao servidor.

CSS — Seletores

CSS define a aparência dos elementos HTML. É vinculado via <link> no head ou dentro de <style>.

/* Seletor de tag — afeta todos os elementos desse tipo */
body {
    background-color: #ddd;
    margin: 0;
    padding: 0;
}

/* Seletor de classe — afeta elementos com class="box" */
.box { border: 1px solid black; }

/* Seletor de id — afeta o elemento com id="titulo" (único na página) */
#titulo { color: blue; }

/* Seletor descendente — links dentro de nav */
nav a {
    color: #fff;
    text-decoration: none;  /* remove o sublinhado */
    font-weight: bold;
    padding: 10px 20px;
}

/* Pseudo-classe :hover — efeito ao passar o mouse */
nav a:hover {
    background-color: #444;
}

/* Múltiplos seletores ao mesmo tempo */
header, footer {
    background-color: #333;
    color: #fff;
    padding: 10px 20px;
}

CSS — Box Model

Todo elemento HTML é uma caixa. O Box Model define como o espaço é calculado:

.elemento {
    width:   200px;    /* largura do conteúdo */
    height:  100px;    /* altura do conteúdo */
    padding: 10px;     /* espaço INTERNO (entre conteúdo e borda) */
    border:  2px solid black; /* borda */
    margin:  20px;     /* espaço EXTERNO (entre este elemento e os vizinhos) */
}

/* box-sizing: border-box faz o padding ser contado DENTRO da width,
   evitando que o elemento fique maior que o esperado. */
* { box-sizing: border-box; }

CSS — Flexbox

Flexbox é usado para organizar elementos em linha ou coluna de forma fácil.

.container {
    display: flex;                /* ativa o flexbox */
    flex-direction: row;          /* row (padrão) = horizontal | column = vertical */
    justify-content: space-between; /* distribui os filhos na direção principal */
    align-items: center;          /* centraliza na direção perpendicular */
    gap: 10px;                    /* espaço entre os filhos */
}

.filho {
    flex: 1;  /* cada filho ocupa espaço igual */
}

PHP — Variáveis e Tipos

PHP é executado no servidor. Todo código fica entre <?php ... ?>. Variáveis começam com $ e não precisam declarar tipo (tipagem dinâmica).

<?php

// Variáveis — o PHP descobre o tipo sozinho
$texto  = "Olá, mundo";   // string
$numero = 42;              // int (inteiro)
$preco  = 9.99;            // float (decimal)
$ativo  = true;            // bool (verdadeiro/falso)
$nada   = null;            // null (sem valor)

// TypeCast — força a conversão de tipo
$inteiro = (int)   "10abc";  // resultado: 10
$decimal = (float) "3.14x"; // resultado: 3.14
$bool    = (bool)  0;        // resultado: false  (0 é false, qualquer outro número é true)

// var_dump — mostra tipo E valor (ferramenta de debug)
var_dump($inteiro);  // int(10)
var_dump($ativo);    // bool(true)

// Constante — valor fixo que não muda durante a execução
define("TAXA_JUROS", 0.02);
echo TAXA_JUROS;  // 0.02  — constante NÃO usa $

?>

PHP — Loops (Laços de Repetição)

<?php

// ── FOR — quando você sabe quantas vezes vai repetir ─────────────────
for ($i = 1; $i <= 10; $i++) {
    echo "$i x 5 = " . ($i * 5) . "<br>";
}
//   ↑ início  ↑ condição    ↑ incremento a cada iteração

// ── FOR aninhado — loops dentro de loops (ex: tabuada) ───────────────
for ($i = 1; $i <= 10; $i++) {
    echo "<h3>Tabuada do $i</h3>";
    for ($j = 1; $j <= 10; $j++) {
        echo "$i x $j = " . ($i * $j) . "<br>";
    }
}

// ── WHILE — repete ENQUANTO a condição for verdadeira ────────────────
$i = 0;
while ($i < 5) {
    echo $i . "<br>";
    $i++;   // cuidado: sem isso vira loop infinito
}

// ── FOREACH — para percorrer ARRAYS ─────────────────────────────────
$frutas = ["maçã", "banana", "uva"];

foreach ($frutas as $fruta) {
    echo $fruta . "<br>";
}

// Com key => value (índice e valor)
foreach ($frutas as $indice => $fruta) {
    echo "$indice: $fruta <br>";
}

// Sintaxe alternativa do foreach (útil misturado com HTML)
foreach ($frutas as $fruta): ?>
    <li><?= $fruta ?></li>
<?php endforeach; ?>

// ── Referência &$ — modifica o array original dentro do loop ─────────
$vetor = [1, 2, 6, 4];
foreach ($vetor as &$valor) {     // & = passa por referência
    if ($valor == 6) $valor++;   // altera o 6 para 7 no array original
}

PHP — $_GET e $_POST

São superglobais — variáveis especiais do PHP que ficam disponíveis em qualquer arquivo.

<?php

// ── $_GET — dados vindos da URL ───────────────────────────────────────
// URL: index.php?pagina=sobre&id=5
$pagina = $_GET["pagina"];     // "sobre"
$id     = $_GET["id"];         // "5"

// ?? — null coalesce: usa o valor se existir, senão usa o padrão
$pagina = $_GET["pagina"] ?? "home";


// ── $_POST — dados vindos de um formulário (method="post") ────────────
// <form method="post"> envia os dados no corpo da requisição (não aparece na URL)
$nome  = $_POST["nome"];
$email = $_POST["email"];

// Sempre verifique se o formulário foi enviado antes de usar $_POST
if (isset($_POST["nome"])) {
    $nome = $_POST["nome"];
}

?>

PHP — include / require

Permitem dividir o site em arquivos menores e reutilizáveis (modularização).

<?php

// include — inclui o arquivo; se não achar, dá WARNING e continua
include "./header.php";

// require — inclui o arquivo; se não achar, dá FATAL ERROR e para tudo
require "./nav.php";

// include_once / require_once — garantem que o arquivo seja incluído APENAS UMA VEZ
// Útil quando vários arquivos incluem o mesmo helper
require_once "./helpers/functions.php";
include_once "./footer.php";

?>

Use require_once para arquivos de funções/helpers (críticos). Use include para partes visuais como header e footer (não críticos).

PHP — match() (PHP 8+)

O match é uma alternativa moderna ao switch. Compara com === (tipo + valor) e retorna um valor diretamente. 

<?php

$pagina = "sobre";

// SWITCH (forma antiga)
switch ($pagina) {
    case "home":
        $arquivo = "pages/home.php";
        break;
    case "sobre":
        $arquivo = "pages/sobre.php";
        break;
    default:
        $arquivo = "pages/404.php";
}

// MATCH (forma moderna — mais curta, sem break, sem "queda" de case)
$arquivo = match($pagina) {
    "home"    => "pages/home.php",
    "sobre"   => "pages/sobre.php",
    "contato" => "pages/contato.php",
    default   => "pages/404.php",   // obrigatório se não cobrir todos os casos
};

// Uso direto com require_once (padrão do projeto)
require_once match($pagina) {
    "home" => "./paginas/home.php",
    "form" => "./paginas/formulario.php",
    "tab"  => "./paginas/tabela.php",
    default => "./paginas/404.php",
};

// Match também pode ser usado com echo
echo "Resultado: " . match(10 % 2) {
    0 => "Par",
    default => "Ímpar"
};

?>
match vs switch: Se o valor não casar com nenhum case e não tiver default, o match lança um UnhandledMatchError. Sempre inclua o default.

PHP — Arrow Functions (fn) (PHP 7.4+)

Forma curta de escrever funções anônimas. Muito usadas com array_map, array_filter, etc.

<?php

// Função anônima tradicional
$dobro = function($n) {
    return $n * 2;
};

// Arrow function — mesma coisa, bem mais curta
// fn(parâmetros) => expressão_de_retorno
$dobro = fn($n) => $n * 2;

echo $dobro(5);   // 10


// Uso prático — array_map + arrow function
$chaves = ["nome", "email", "senha"];

[$nome, $email, $senha] = array_map(
    fn($key) => validaInput($_POST[$key]),
    $chaves
);
// Para cada chave, pega o valor do POST e passa pelo validaInput()
// O resultado é desestruturado nas variáveis $nome, $email, $senha

?>

function calcularIdade($data): int {

    // Cria um objeto DateTime com a data de nascimento
    $nascimento = new DateTime($data);

    $agora = new DateTime();

    return $agora->diff($nascimento)->y;

PHP — Segurança e Validação de Dados

<?php

// ── validaInput — função que limpa dados recebidos ─────────────────────
// SEMPRE use antes de exibir dados vindos do usuário na tela!
function validaInput($dados) {

    // trim() — remove espaços do início e do fim
    // Ex: "  Max  " → "Max"
    $dados = trim($dados);

    // htmlspecialchars() — converte caracteres especiais em entidades HTML
    // Ex: <script> vira &lt;script&gt; (não é executado)
    // Proteção contra XSS (Cross-Site Scripting)
    $dados = htmlspecialchars($dados);

    return $dados;
}

// Uso prático
$nome = validaInput($_POST["nome"]);


// ── Verificar se formulário foi enviado antes de processar ─────────────
if (isset($_POST["nome"])) {
    // processa os dados
}


// ── Validar se senha == confirmação ───────────────────────────────────
$senhaConfere = ($senha === $confSenha) ? "Sim" : "Não";
// Usa === (triplo) para comparar tipo + valor


// ── ?? (null coalesce) — evita erro ao acessar variável que pode não existir
?>
<td><?= $nome ?? "" ?></td>
<!-- Se $nome existir, mostra; se não, mostra string vazia -->
NUNCA exiba diretamente na tela o que veio do usuário sem passar por htmlspecialchars() ou validaInput(). Um usuário malicioso pode injetar código JavaScript na página (ataque XSS).

Padrão — Roteamento com $_GET + match()

Este é o padrão usado em todo o projeto para criar um "mini-sistema" sem framework. Um único index.php lê a URL e carrega o arquivo correto.

<a href="?page=home">Home</a>
<a href="?page=sobre">Sobre</a>
<a href="?page=contato">Contato</a>
<?php
// ── index.php — o "controlador" ──────────────────────────────────────

// 1. Lê o parâmetro da URL. Se não existir, usa "home" como padrão
$pagina = $_GET["page"] ?? "home";

?>
<!DOCTYPE html>
<html><head>...</head><body>
<?php

// 2. Inclui partes fixas do layout
require_once "./header.php";

Padrão — Arquivo de Funções (helpers/functions.php)

Centralizar as funções em um arquivo separado evita repetição e facilita a reutilização. Inclua com require_once nos arquivos que precisarem. 

<?php
// helpers/functions.php

// ── validaInput — limpa dados do usuário ──────────────────────────────
function validaInput($dados) {
    $dados = trim($dados);           // remove espaços extras
    $dados = htmlspecialchars($dados);  // protege contra XSS
    return $dados;
}


// ── calcularIdade — calcula anos a partir de data de nascimento ────────
function calcularIdade($data): int {
    $nascimento = new DateTime($data);   // cria objeto com a data recebida
    $agora      = new DateTime();          // cria objeto com data atual
    return $agora->diff($nascimento)->y;  // diferença em anos
}


// ── periodoDia — retorna Manhã, Tarde ou Noite baseado na hora ─────────
function periodoDia($hora): string {

    // explode(":","14:30") → ["14","30"]
    // [0] pega apenas a hora inteira
    $hora = explode(":", $hora)[0];

    if     ($hora >= 0  && $hora < 12) return "Manhã";
    elseif ($hora >= 12 && $hora < 18) return "Tarde";
    else                                  return "Noite";
}


// ── calcularEmprestimo — juros simples ────────────────────────────────
define("TAXA_JUROS", 0.02);  // 2% ao mês (constante global)

function calcularEmprestimo(float $valor, int $meses): float {
    // Fórmula: M = P * (1 + i * n)
    return $valor * (1 + (TAXA_JUROS * $meses));
}

// Formatar resultado em moeda BR: number_format(valor, decimais, sep_decimal, sep_milhar)
echo "R$ " . number_format(calcularEmprestimo(1000, 12), 2, ',', '.');
// R$ 1.240,00

?>

PHP — Funções Anônimas (Closures)

Uma função anônima (também chamada de closure) é uma função sem nome que pode ser atribuída a uma variável, passada como argumento ou usada diretamente no código. É muito útil quando você precisa de uma função rápida e temporária.

Características

CaracterísticaDetalhe
Sem nomefunction () {} — não possui identificador
ArmazenávelPode ser salva em uma variável
Variáveis externasAcessa o escopo externo via use
PassávelPode ser passada como parâmetro para outras funções

Estrutura básica

$variavel = function ($parametros) use ($variaveis_externas) {
    // código
    return $valor;
};

// $variavel       → onde a função é armazenada
// function        → palavra-chave
// ($parametros)   → parâmetros normais
// use (...)       → opcional: importa variáveis do escopo externo

Exemplo com use

$desconto = 0.1;

$aplicarDesconto = function ($preco) use ($desconto) {
    return $preco - ($preco * $desconto);
};

echo $aplicarDesconto(100); // 90
use vs parâmetro: use captura variáveis do escopo onde a função foi criada. Parâmetros recebem valores no momento da chamada.

PHP — array_map

O array_map percorre um array e aplica uma função em cada elemento, retornando um novo array com os resultados — sem alterar o original.

Estrutura básica

// Com função anônima:
$novoArray = array_map(function ($item) {
    return $resultado;
}, $array);

// array_map()       → função principal
// function ($item)  → aplicada em cada elemento
// $item             → valor atual do array
// return            → novo valor no array resultante
// $array            → array original (não é alterado)

Exemplos

// ── Dobrar números ────────────────────────────────────────────────
$numeros = [1, 2, 3, 4];
$resultado = array_map(function ($n) {
    return $n * 2;
}, $numeros);
// [2, 4, 6, 8]

// ── Com variável externa (use) ────────────────────────────────────
$taxa = 2;
$resultado = array_map(function ($n) use ($taxa) {
    return $n * $taxa;
}, [1, 2, 3]);
// [2, 4, 6]

// ── Com strings ───────────────────────────────────────────────────
$nomes = ["joão", "maria", "pedro"];
$maiusculas = array_map(function ($nome) {
    return strtoupper($nome);
}, $nomes);
// ["JOÃO", "MARIA", "PEDRO"]

Versão moderna — Arrow Function

// Sintaxe mais curta com fn() =>
$numeros = [1, 2, 3];
$resultado = array_map(fn($n) => $n * 2, $numeros);
// [2, 4, 6]

// Arrow functions capturam variáveis externas automaticamente (sem 'use')
$taxa = 3;
$resultado = array_map(fn($n) => $n * $taxa, [1, 2, 3]);
// [3, 6, 9]
Quando usar array_map: quando quiser transformar todos os elementos de um array aplicando a mesma lógica, sem alterar o original e de forma legível.
array_map vs array_filter vs array_reduce: map transforma elementos  |  filter filtra elementos  |  reduce reduz a um único valor.
